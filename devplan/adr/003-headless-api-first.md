# ADR-003: 헤드리스 코어 + API 우선

> 상태: 확정 · 날짜: 2026-07-04

## 컨텍스트
웹/모바일/메신저/CI 등 **다양한 클라이언트**가 같은 기능을 사용해야 한다. Cursor IDE API는 없다.

## 결정
- 비즈니스 로직은 **헤드리스 코어**에 둔다.
- **자체 `/api/v1`** 이 유일한 기능 계약이다.
- 웹 프론트는 특권 없는 클라이언트다.

## 근거
- 메신저·온프렘 확장 시 코어 변경 없이 어댑터만 추가(포트&어댑터).
- 테스트·자동화·CI가 API만 호출하면 된다.

## 대안
- **웹에 로직 집중**: 메신저/CI 재구현·불일치.
- **Cursor IDE relay**: "그 컴퓨터 없음" 전제와 충돌.

## 결과
Command 처리기(17) 단일 진입, 02 API 버저닝.
