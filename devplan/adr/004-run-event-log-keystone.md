# ADR-004: RunEventLog 키스톤

> 상태: 확정 · 날짜: 2026-07-04

## 컨텍스트
모바일에서 연결이 끊겨도 run을 관찰해야 하고, 인박스·알림·감사·리플레이가 필요하다.

## 결정
모든 run 이벤트를 **저장 후 발행**(append-then-publish). RunEventLog(06)가 관찰성의 단일 원천이다.

## 근거
- WS는 구독자 중 하나일 뿐, run 수명과 분리(SR-06).
- 재접속 리플레이·다채널 팬아웃(웹훅·메신저)이 동일 이벤트에서 파생.

## 대안
- **WS만 스트리밍, DB 미저장**: 재접속·감사 불가.
- **폴링 run 상태**: 세밀한 tool/plan 스트림 불가.

## 결과
P0 PoC부터 RunEventLog 검증. 상태 전이도 이벤트 소비로 유도(ADR-008).
