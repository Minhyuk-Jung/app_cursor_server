# ADR-008: 상태 전이 RunEventLog 단일 원천

> 상태: 확정 · 날짜: 2026-07-04

## 컨텍스트
SessionManager가 상태머신을 직접 호출하고 RunEventLog가 이벤트를 소비하면 **이중 전이** 위험이 있다(R-06).

## 결정
- 상태 전이는 **RunEventLog에 append된 DomainEvent를 상태머신이 소비**할 때만 발생한다.
- SessionManager는 이벤트를 **기록만** 하고 상태머신을 직접 호출하지 않는다.

## 근거
- 단일 원천 → 인박스·알림·상태 조회 일관.
- 이벤트 로그와 상태 DB 정합 추적 용이.

## 대안
- **SessionManager 직접 호출**: 이중 트리거·경합.
- **상태만 DB, 이벤트 선택적**: 리플레이와 상태 불일치.

## 결과
05/06/07 문구 통일. R-06 닫힘.
